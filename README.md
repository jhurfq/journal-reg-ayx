# journal-reg-ayx

本仓库用于归档与发布多个独立的 HTML 页面。这些页面在本地编写或整理后，通过 GitHub Pages 进行托管与展示。

## 项目简介

这是一个以静态页面归档为目的的仓库，不针对任何特定的网站或域名。页面内容涵盖日常记录、工具页面、资料整理等类型，每个页面均为独立文件，不依赖后端服务。

## 目录说明

```
.
├── index.html          # 仓库主页面（可选）
├── page1.html          # 示例归档页面
├── page2.html
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── style.css
│   └── script.js
└── README.md           # 本文件
```

- 根目录下的 `.html` 文件为可直接访问的归档页面。
- `assets/` 目录存放页面共用的样式、脚本或图片资源。
- 如有多个子目录，用于按主题或日期归类。

## 页面归档说明

- 每个 HTML 页面保持独立，可单独通过 URL 访问。
- 页面之间无强制链接关系，但可通过导航菜单串联。
- 所有页面均为静态内容，无需服务器端支持。
- 页面内容不包含任何商业推广、外部跳转或诱导点击信息。

## 维护说明

- 新增页面：在根目录或合适子目录下添加 `.html` 文件，并更新 `index.html` 中的列表（如有）。
- 修改页面：直接编辑对应 HTML 文件，注意保持资源路径正确。
- 删除页面：移除文件后，建议同步更新索引页面。
- 本仓库不自动生成页面索引，请手动维护导航或列表。

## 使用方式

1. 克隆本仓库到本地。
2. 在本地编辑或添加 HTML 文件。
3. 提交并推送至 GitHub 主分支。
4. 启用 GitHub Pages（Settings > Pages > Source 选择 main 分支）。
5. 通过 `https://<用户名>.github.io/journal-reg-ayx/` 访问。
